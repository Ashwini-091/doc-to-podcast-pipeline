================================================================================
PIPELINE ARCHITECTURE
================================================================================

INPUT:  Document (PDF, DOCX, TXT)
  ↓
PROCESSING:
  1. Extract text from document
  2. Split into chunks (512 tokens, 50-token overlap)
  3. Generate embeddings (Sentence Transformers)
  4. Store in FAISS index
  ↓
GENERATION (Plan → Expand → Critique → Revise):
  1. Planning Phase
     - Read full document
     - Create podcast outline (5 sections)
  2. Expansion Phase (per section)
     - Retrieve relevant chunks using FAISS
     - Write section using retrieved context
  3. Criticism Phase
     - Review script against source document
     - Identify unsupported claims
  4. Revision Phase (if needed)
     - Fix unsupported claims
     - Loop back to Criticism (max 3 times)
  ↓
AUDIO CONVERSION:
  1. Convert final script to speech (gTTS or pyttsx3)
  2. Generate MP3 audio file
  ↓
OUTPUT: Transcript (TXT) + Audio (MP3)

SUPPORT & TROUBLESHOOTING
================================================================================

Problem: "ModuleNotFoundError"
Solution: pip install -r requirements.txt --force-reinstall

Problem: "ANTHROPIC_API_KEY not set"
Solution: Make sure you set it in your terminal (not .env)

Problem: "FAISS index error"
Solution: rm -rf data/vectors/ then rebuild in app

Problem: "gTTS fails (no internet)"
Solution: Switch to pyttsx3 in sidebar configuration
